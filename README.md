# Pico Sync Tool

**Pico Sync** — компактний CLI-інструмент для зручної роботи з Raspberry Pi Pico / Pico W під MicroPython.
Надає команди для синхронізації файлів, перегляду файлової системи, базових операцій з файлами на пристрої,
моніторингу логів та перезавантаження плати.

Працює поверх `mpremote`. Підтримує інтерактивний режим з меню, fzf, вибір мови (UA/EN).

---

## Встановлення

```bash
pip install pico_sync
```

Або з GitHub:

```bash
git clone https://github.com/toldk98/pico_sync.git
cd pico_sync
pip install -r requirements.txt
```

### Залежності

- Python 3.7+
- `pyserial` — робота з serial-портами
- `mpremote` — виконання команд на Pico

### Опціональні залежності

- **`fzf`** — швидкий пошук і зручне меню замість нумерованого вводу. Не впливає на роботу pico_sync, але значно покращує UX:

  | Система | Встановлення |
  |---------|-------------|
  | Linux (Debian/Ubuntu) | `sudo apt install fzf` |
  | Linux (Arch) | `sudo pacman -S fzf` |
  | Linux (Fedora) | `sudo dnf install fzf` |
  | macOS | `brew install fzf` |
  | Windows | `scoop install fzf` |
  | Будь-яка | Завантажити binary: https://github.com/junegunn/fzf |

  Після встановлення перезапустіть pico_sync — fzf підхопиться автоматично.

## 1. Призначення

Мета інструмента — зробити розробку на MicroPython максимально схожою на роботу з локальним проєктом.

**Можливості:**

- 🔄 **Синхронізація** — завантаження файлів з `src/` на Pico (див. [Інтерактивний режим](#6-інтерактивний-режим) або [CLI](#7-cli-довідка-та-приклади))
- 🧠 **SHA-синхронізація** — оновлюються лише змінені файли (SHA-256)
- 🗑 **Автовидалення зайвих файлів** — файли на Pico, яких немає локально, видаляються
- 🧼 **Очищення порожніх директорій** — після синхронізації
- ⛔ **`.picoignore`** — ігнорування файлів (як `.gitignore`)
- 📂 **Інтерактивний браузер файлів** — перегляд, читання, редагування, видалення на Pico
- 🔍 **Авто-детект порту** — пошук Pico серед serial-портів
- 🔌 **Серійний монітор** — живий перегляд логів з Pico
- 🔁 **Перезавантаження** — програмний reset плати
- 🌐 **Мова інтерфейсу** — Українська / English
- 📦 **Менеджмент проектів** — збереження проектів з налаштуваннями

Інструмент займає ~246 КБ, розбитий на модулі для зручності супроводу та тестування. За бажанням можна видаляти окремі модулі, які не потрібні.

## 3. Структура проєкту

```
project_root/
│
├── .picoignore           # правила ігнорування
├── .picosyncconfig       # конфігурація проекту (порт, фільтр)
├── meta/                 # мета-дані (оновлення тощо)
│
└── src/                  # файли, які синхронізуються з Pico
```

⚠️ **Важливо:**
1. `.picoignore` працює тільки для файлів всередині `src/`
2. Синхронізується виключно `src/` — файли за межами ніколи не потрапляють на Pico

## 4. Файл .picoignore

Правила ігнорування. Формат аналогічний `.gitignore`.

**Приклади:**

```
# Ігнорувати кеші
__pycache__/
*.pyc

# Рекурсивно всі .log
**/*.log

# Конкретний файл
config/local_settings.py

# Директорія
tests/
```

**Підтримуються патерни:**

| Патерн | Опис |
|--------|------|
| `*` | Будь-які символи в межах директорії |
| `**` | Рекурсивний матч (будь-який рівень) |
| `?` | Один символ |
| `dir/` | Вся директорія |
| `*.ext` | Маска файлів |

## 5. Технічні деталі

### SHA-синхронізація

1. Для кожного локального файлу обчислюється SHA-256
2. На Pico через один mpremote exec обчислюються SHA-256 всіх віддалених файлів
3. Якщо хеші збігаються → файл пропускається
4. Якщо хеші різні або файл відсутній → файл завантажується
5. Файли на Pico, яких немає локально, видаляються
6. Порожні директорії очищаються

### Порт

Порт визначається в такому порядку:
1. Якщо в `.picosyncconfig` налаштовано `piconame` — пошук Pico з відповідним вмістом `/.piconame` серед усіх підключених пристроїв
2. Значення `port` з `.picosyncconfig` (якщо налаштовано)
3. Авто-детект: пошук серед підключених serial-пристроїв за USB VID. Якщо знайдено кілька Pico — використовується перший
4. Якщо нічого не знайдено — виводиться повідомлення про відсутній порт

Порт зберігається в `os.environ["MPREMOTE_PORT"]`.

#### .piconame

`/.piconame` — опціональний текстовий файл-ідентифікатор на самому Pico.
Його можна створити вручну або через `picosync --set-name my-device`. Якщо `piconame` налаштовано в конфігу проекту, pico_sync знайде саме той Pico, чиє ім'я збігається, навіть якщо підключено кілька пристроїв.

### BAUD = 115200

Стандартна швидкість серійного порту MicroPython на RP2040.
Використовується в режимі моніторингу (`--monitor`).

## 6. Інтерактивний режим

Без аргументів (або з `--pick`) запускається інтерактивний режим.

При першому запуску ви побачите список збережених проектів. Якщо проектів ще немає — пропозицію додати поточну директорію. Вибравши проект (або додавши новий), ви потрапляєте в головне меню проекту.

### Головне меню

```
..          — повернутись до списку проектів
[i] info    — інформація про проект (лише перегляд)
[f] files   — браузер файлів на Pico
[d] device  — синхронізація, монітор, перезавантаження
[c] config  — налаштування порту, src, оновлень, ініціалізація
```

### Браузер файлів

Дозволяє переглядати дерево файлів на Pico. Для кожного файлу доступні дії:
- `cat` — вивести вміст
- `edit` — редагувати в системному редакторі
- `rm` — видалити
- `[r] refresh` — оновити список файлів
- `[*] find` — пошук по всіх файлах

### Пристрій

- `sync` — синхронізація з вибором фільтра
- `monitor` — серійний монітор
- `reboot` — перезавантаження Pico

### Налаштування

- `port_settings` — налаштування підключення Pico: ручний вибір порту або авто-пошук за іменем
- `check_update` — перевірка оновлень
- `init` — створення `.picoignore`, `meta/`, `.picosyncconfig`

### Мова

У списку проектів доступний пункт `[~] lang` для перемикання мови.

### Фільтри синхронізації

| Фільтр | Опис |
|--------|------|
| `all` | Всі файли (видаляє зайве на Pico) |
| `py` | Тільки `.py` |
| `py+` | `.py`, `.txt`, `.json` |
| `nopy` | Все крім `.py` |
| `.ext,.ext2` | Власні розширення через кому |

### Приклад сеансу

```
$ picosync

  [1] my-project  (/home/user/projects/my-project)
  [+] add project
  [~] lang

> 1                         # вибрали проект

  ..  — back to project list
  [i] info    — інформація про проект
  [f] files   — браузер файлів на Pico
  [d] device  — синхронізація, монітор, перезавантаження
  [c] config  — налаштування

> d                         # меню пристрою

  sync
  monitor
  reboot

> sync                      # синхронізація
  filter (all):
  Зібрано файлів: 12, синхронізовано: 3, пропущено: 9
```

## 7. CLI довідка та приклади

### Довідка

```bash
picosync --help
```

```
usage: picosync [-h] [--port PORT] [--src SRC] [--sync] [--ls PATH]
                [--cat FILE] [--edit FILE] [--search_port] [--check_update]
                [--reboot] [--monitor] [--pick] [--filter FILTER] [--init]
                [--lang {ua,en}] [--version]
                {project} ...

Pico Sync Tool — sync/ls/cat/edit for Raspberry Pi Pico

options:
  -h, --help            Показати довідку
  --port PORT           Pico COM port (авто-детект якщо не вказано)
  --src SRC             Директорія для синхронізації (default: src)
  --sync                Синхронізувати src → Pico
  --ls PATH             Список файлів в директорії на Pico
  --cat FILE            Вивести вміст файлу
  --edit FILE           Редагувати файл
  --search_port         Інтерактивний пошук serial-портів
  --check_update        Перевірити оновлення
  --reboot              Перезавантажити Pico
  --monitor             Серійний монітор
  --pick                Інтерактивний режим
  --filter FILTER       Фільтр синхронізації (default: all)
  --init                Створити .picoignore та meta/
  --set-name NAME       Записати /.piconame на Pico і зберегти в конфіг
  --lang {ua,en}        Мова інтерфейсу
  --version             Версія
```

### Синхронізація

```bash
picosync --sync                          # всі файли
picosync --sync --filter py              # тільки .py
picosync --sync --filter .wav,.mpy       # вказані розширення
picosync --src my_src --sync             # нестандартний src
```

### Перегляд файлів

```bash
picosync --ls /
picosync --ls /spm
picosync --cat /main.py
```

### Редагування

```bash
picosync --edit /config/settings.py
```

Файл завантажується в тимчасовий файл, відкривається в редакторі, після збереження — автоматично завантажується назад на Pico.

### Пошук порту

```bash
picosync --search_port
```

### Монітор

```bash
picosync --monitor
```

Автоматично перепідключається при:
- перезавантаженні Pico
- зміні USB-порту
- тимчасовому відключенні

### Перезавантаження

```bash
picosync --reboot
```

Програмний reset через `mpremote reset` — еквівалент натискання RUN.

### Версія

```bash
picosync --version
```

## 8. Система проектів

Проекти зберігаються в `~/.config/pico_sync/projects.json`.
Кожен проект містить шлях до директорії, налаштування порту, фільтр синхронізації та останній використаний src.

### Керування проектами

```bash
picosync project list          # список проектів
picosync project add /шлях     # додати існуючу директорію
picosync project remove назва  # видалити проект зі списку
```

В інтерактивному режимі проекти відображаються на стартовому екрані.
Пункт `[+] add project` додає поточну директорію, `[~] lang` перемикає мову.

## 10. Поширені питання (FAQ)

### Pico не знайдено / порт не визначено
Переконайтеся що Pico підключено USB-кабелем (не тільки живлення). Якщо авто-детект не спрацював — використайте `picosync --search_port` для ручного вибору.

### Два Pico підключено одночасно
Авто-детект візьме перший знайдений. Щоб працювати з конкретним пристроєм — налаштуйте `.piconame` через `picosync --set-name <name>` або `[c] config → piconame → set`.

### Файли не синхронізуються / не всі файли на Pico
Перевірте `.picoignore` — можливо файл підпадає під правило ігнорування. За замовчуванням синхронізується директорія `src/`.

### Помилка "No module named 'pyserial' / 'mpremote'"
Встановіть залежності: `pip install pyserial mpremote`

### Як оновити Pico Sync?
```bash
pip install --upgrade pico_sync
```
Або `[c] config → check_update` в інтерактивному режимі.
