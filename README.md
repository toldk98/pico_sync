# Pico Sync

Інструмент для синхронізації файлів і роботи з Raspberry Pi Pico / Pico W під MicroPython.

Працює поверх `mpremote` — підключіть Pico через USB і керуйте файлами через термінал.

---

## Встановлення

```bash
pip install pico_sync
```

Або з GitHub:

```bash
git clone https://github.com/toldk98/pico_sync.git
cd pico_sync
pip install -r requirements.txt
```

## Використання

Інтерактивний режим (рекомендовано):

```bash
picosync
```

Або через модуль:

```bash
python -m pico_sync
```

## Можливості

| Команда | Опис |
|---------|------|
| `--pick` | Інтерактивний режим з меню проєктів |
| `--sync` | Синхронізувати `src/` на Pico |
| `--ls PATH` | Список файлів на Pico |
| `--cat FILE` | Показати вміст файлу |
| `--nano FILE` | Редагувати файл через nano |
| `--monitor` | Серійний монітор |
| `--search_port` | Інтерактивний вибір порту |
| `--reboot` | Перезавантажити Pico |
| `--filter` | Фільтр синхронізації (all, py, py+, nopy, .ext) |
| `--lang {ua,en}` | Мова інтерфейсу |
| `--version` | Версія |
| `--init` | Створити .picoignore та meta/ |

## Інтерактивний режим

`picosync` (без аргументів) запускає інтерактивний режим з:

- **Список проєктів** — додавати/видаляти проєкти, перемикати мову
- **Інформація про проєкт** — пристрій, порт, фільтр (read-only)
- **Файловий браузер** — перегляд, читання, редагування, видалення файлів на Pico
- **Пристрій** — синхронізація з вибором фільтра, монітор, перезавантаження
- **Налаштування** — порт, src-директорія, перевірка оновлень, ініціалізація

Якщо в системі є `fzf` — використовується він як меню. Інакше — нумерований вибір.

## Структура проєкту

```
project_root/
├── .picoignore        # правила ігнорування
├── .picosyncconfig    # конфігурація проєкту
├── meta/              # мета-дані (оновлення тощо)
└── src/               # файли для синхронізації
```

## Конфігурація

`.picoignore` — формат як `.gitignore`. Підтримка `*`, `**`, `?`, `dir/`, `*.ext`.

`.picosyncconfig` — автоматично створюється через `[c] config → init`:
```json
{"port": "/dev/ttyACM0", "filter": "all"}
```

Фільтри синхронізації:
- `all` — всі файли (видаляє зайве на Pico)
- `py` — тільки .py
- `py+` — .py, .txt, .json
- `nopy` — все крім .py
- власні розширення через кому
