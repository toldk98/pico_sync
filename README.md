# Pico Sync Tool

**Pico Sync** — компактний CLI-інструмент для зручної роботи з Raspberry Pi Pico / Pico W під MicroPython.
Надає команди для синхронізації файлів, перегляду файлової системи, базових операцій з файлами на пристрої,
моніторингу логів та перезавантаження плати.

Працює поверх `mpremote`. Підтримує інтерактивний режим з меню, fzf, вибір мови (UA/EN).

---

## Встановлення

```bash
pip install pico_sync
```

Або з GitHub:

```bash
git clone https://github.com/toldk98/pico_sync.git
cd pico_sync
pip install -r requirements.txt
```

### Залежності

- Python 3.7+
- `pyserial` — робота з serial-портами
- `mpremote` — виконання команд на Pico
- `fzf` (опціонально) — швидкий пошук і меню

## 📂 2. Структура проєкту

```
project_root/
│
├── .picoignore           # правила ігнорування
├── .picosyncconfig       # конфігурація проекту (порт, фільтр)
├── meta/                 # мета-дані (оновлення тощо)
│
└── src/                  # файли, які синхронізуються з Pico
```

⚠️ **Важливо:**
1. `.picoignore` працює тільки для файлів всередині `src/`
2. Синхронізується виключно `src/` — файли за межами ніколи не потрапляють на Pico

## 📜 3. Файл .picoignore

Правила ігнорування. Формат аналогічний `.gitignore`.

**Приклади:**

```
# Ігнорувати кеші
__pycache__/
*.pyc

# Рекурсивно всі .log
**/*.log

# Конкретний файл
config/local_settings.py

# Директорія
tests/
```

**Підтримуються патерни:**

| Патерн | Опис |
|--------|------|
| `*` | Будь-які символи в межах директорії |
| `**` | Рекурсивний матч (будь-який рівень) |
| `?` | Один символ |
| `dir/` | Вся директорія |
| `*.ext` | Маска файлів |

## 4. 🔧 Технічні деталі

### Delta-sync (SHA-256)

1. Для кожного локального файлу обчислюється SHA-256
2. На Pico через один mpremote exec обчислюються SHA-256 всіх віддалених файлів
3. Якщо хеші збігаються → файл пропускається
4. Якщо хеші різні або файл відсутній → файл завантажується
5. Файли на Pico, яких немає локально, видаляються
6. Порожні директорії очищаються

### Порт

Порт визначається автоматично. При першому запуску:
1. Перевіряється `.picosyncconfig`
2. Якщо порт не налаштовано — авто-детект серед підключених пристроїв
3. Порт зберігається в `os.environ["MPREMOTE_PORT"]`

### BAUD = 115200

Стандартна швидкість серійного порту MicroPython на RP2040.
Використовується в режимі моніторингу (`--monitor`).

## 5. Інтерактивний режим

Без аргументів (або з `--pick`) запускається інтерактивний режим.

### Головне меню

```
..          — повернутись до списку проектів
[i] info    — інформація про проект (лише перегляд)
[f] files   — браузер файлів на Pico
[d] device  — синхронізація, монітор, перезавантаження
[c] config  — налаштування порту, src, оновлень
```

### Браузер файлів

Дозволяє переглядати дерево файлів на Pico. Для кожного файлу доступні дії:
- `cat` — вивести вміст
- `edit` — редагувати в системному редакторі
- `rm` — видалити
- `[r] refresh` — оновити список файлів
- `[*] find` — пошук по всіх файлах

### Пристрій

- `sync` — синхронізація з вибором фільтра
- `monitor` — серійний монітор
- `reboot` — перезавантаження Pico

### Налаштування

- `port` — вибір COM-порту
- `src` — зміна директорії синхронізації
- `check_update` — перевірка оновлень
- `init` — створення `.picoignore`, `meta/`, `.picosyncconfig`

### Мова

У списку проектів доступний пункт `[~] lang` для перемикання мови.

### Фільтри синхронізації

| Фільтр | Опис |
|--------|------|
| `all` | Всі файли (видаляє зайве на Pico) |
| `py` | Тільки `.py` |
| `py+` | `.py`, `.txt`, `.json` |
| `nopy` | Все крім `.py` |
| `.ext,.ext2` | Власні розширення через кому |

## 6. CLI довідка та приклади

### Довідка

```bash
picosync --help
```

```
usage: picosync [-h] [--port PORT] [--src SRC] [--sync] [--ls PATH]
                [--cat FILE] [--edit FILE] [--search_port] [--check_update]
                [--reboot] [--monitor] [--pick] [--filter FILTER] [--init]
                [--lang {ua,en}] [--version]
                {project} ...

Pico Sync Tool — sync/ls/cat/edit for Raspberry Pi Pico

options:
  -h, --help            Показати довідку
  --port PORT           Pico COM port (авто-детект якщо не вказано)
  --src SRC             Директорія для синхронізації (default: src)
  --sync                Синхронізувати src → Pico
  --ls PATH             Список файлів в директорії на Pico
  --cat FILE            Вивести вміст файлу
  --edit FILE           Редагувати файл
  --search_port         Інтерактивний пошук serial-портів
  --check_update        Перевірити оновлення
  --reboot              Перезавантажити Pico
  --monitor             Серійний монітор
  --pick                Інтерактивний режим
  --filter FILTER       Фільтр синхронізації (default: all)
  --init                Створити .picoignore та meta/
  --lang {ua,en}        Мова інтерфейсу
  --version             Версія
```

### Синхронізація

```bash
picosync --sync                          # всі файли
picosync --sync --filter py              # тільки .py
picosync --sync --filter .wav,.mpy       # вказані розширення
picosync --src my_src --sync             # нестандартний src
```

### Перегляд файлів

```bash
picosync --ls /
picosync --ls /spm
picosync --cat /main.py
```

### Редагування

```bash
picosync --edit /config/settings.py
```

Файл завантажується в тимчасовий файл, відкривається в редакторі, після збереження — автоматично завантажується назад на Pico.

### Пошук порту

```bash
picosync --search_port
```

### Монітор

```bash
picosync --monitor
```

Автоматично перепідключається при:
- перезавантаженні Pico
- зміні USB-порту
- тимчасовому відключенні

### Перезавантаження

```bash
picosync --reboot
```

Програмний reset через `mpremote reset` — еквівалент натискання RUN.

### Версія

```bash
picosync --version
```
